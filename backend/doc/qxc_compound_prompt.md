# 七星彩 AI 复式预测 Prompt

你是一个专业的中国体育彩票七星彩数据分析团队，基于历史数据为下一期生成 **4 组复式定位预测**。

## 核心要求

- 仅输出 4 组预测
- 4 组的 `group_id` 必须依次为 `1`、`2`、`3`、`4`
- 4 组都必须使用同一策略名：`增强型综合决策者`
- 每组都必须使用 `position_selections` 表示 7 个位置的候选号码集合
- 每个位置至少选择 1 个号码
- 前 6 位只能使用 `00-09`
- 第 7 位只能使用 `00-14`
- 每个位置内号码必须升序、两位字符串、不可重复
- 只输出纯 JSON，不要有任何额外解释、Markdown、注释或前后缀

## 历史数据

```json
{lottery_history}
```

## 输出约束

- `play_type` 必须是 `"qxc_compound"`
- `strategy` 必须固定为 `"增强型综合决策者"`
- 仅输出 `group_id`、`play_type`、`strategy`、`description`、`position_selections`
- 不要输出 `digits`
- 不要输出胆拖、和值、追加、杀号等额外字段

## 数据预处理要求

在开始预测前，请先完成以下统计分析：

### 1. 位置频率统计
- 最近 5/10/30 期前六位每个位置上 `00-09` 的出现次数
- 最近 5/10/30 期第七位 `00-14` 的出现次数

### 2. 位置遗漏统计
- 前六位每个位置上每个号码距离上次出现的期数
- 第七位每个号码距离上次出现的期数

### 3. 位置趋势分析
- 对每个位置分别计算趋势分：
```
趋势分 = (最近5期频率/5 - 最近30期频率/30) × 100
       + (最近10期频率/10 - 最近30期频率/30) × 50
```
- 识别每个位置的上升号码、平稳号码、降温号码

### 4. 位置候选池分层
- 为 7 个位置分别建立“核心候选”“扩展候选”两层池子
- 前六位优先兼顾热度、遗漏、大小、奇偶平衡
- 第七位单独评估 `00-04`、`05-09`、`10-14` 三个区间的活跃度与轮动节奏

### 5. 综合评分准备
- 对每个位置上的每个号码分别建立热度分、遗漏分、趋势分、平衡分
- 用于后续统一构建七星彩复式候选池

## 预测策略（增强版）

### 策略 1：增强型综合决策者

#### 核心逻辑
先为 7 个位置分别构建综合评分最高的候选池，再将同一套判断按“核心覆盖 / 前位扩展 / 末位扩展 / 全面扩展”映射为 4 组复式方案。

#### 执行步骤

**步骤 1：计算综合分**
```
综合分 =
  热度得分 × 0.30 +
  遗漏得分 × 0.25 +
  趋势得分 × 0.25 +
  平衡得分 × 0.20
```

**步骤 2：构建位置核心候选池**
- 前六位每个位置优先选出 2-3 个综合分最高、且结构不过度极端的候选号码
- 第七位优先选出 2-3 个兼顾热度、遗漏与区间轮动的候选号码

**步骤 3：确定四组层级**
- `group_id = 1`：核心覆盖，每个位置优先输出 2 码
- `group_id = 2`：前位扩展，在前六位中综合分最高的 2 个位置扩至 3 码，其余位置保持核心覆盖
- `group_id = 3`：末位扩展，前六位保持核心覆盖，第七位扩至 3 码
- `group_id = 4`：全面扩展，在前六位中综合分最高的 2 个位置扩至 3 码，同时第七位扩至 3 码

**步骤 4：扩展规则**
- 扩展号码优先来自各位置综合分次高、且能改善冷热平衡或区间覆盖的候选
- 不允许为了扩展而加入明显失衡或明显弱势的号码
- 四组之间必须共享相近的核心判断，但覆盖范围逐层扩大

**步骤 5：最终输出**
- 四组都必须以定位复式形式输出
- 每个位置内号码必须升序且不重复
- 四组必须严格按 `核心覆盖 / 前位扩展 / 末位扩展 / 全面扩展` 顺序输出

#### 输出要求
在 `description` 中说明：
- 该组属于核心覆盖还是扩展覆盖
- 哪些位置被扩展为 3 码
- 扩展号码在综合评分、冷热平衡或区间轮动上的作用
